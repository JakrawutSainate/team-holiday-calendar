# เอกสารระบบ HolidayHQ - ระบบปฏิทินวันหยุดและจัดการสิทธิ์ของทีม (Team Holiday Calendar System)

ระบบ HolidayHQ เป็นเว็บแอปพลิเคชันที่ออกแบบมาสำหรับการบริหารจัดการเวรปฏิบัติงาน (Shift), สิทธิ์การลาพักผ่อน และระบบโทเค็นสะสมของทีมงาน โดยมีคุณสมบัติที่สำคัญดังต่อไปนี้

---

## 1. คุณสมบัติหลักของระบบ (Core Features)

### 1.1 หน้าภาพรวมแดชบอร์ด (Overview Dashboard)
- **operatingCapacity**: แสดงสถานะและภาระงานปัจจุบันของทีมงาน
- **Upcoming Holidays**: รายการแสดงรายชื่อสมาชิกทีมที่มีกำหนดการลาหรือหยุดงานในอนาคตอันใกล้ พร้อมรูปโปรไฟล์และจำนวนวันหยุด
- **Team Availability**: แผนภูมิวัดประสิทธิภาพของทีมว่า ณ ปัจจุบันมีบุคลากรพร้อมทำงานกี่เปอร์เซ็นต์ (Active / Present)
- **Token Balance Card**: การแสดงยอดเงินโทเค็นสะสมของสมาชิก ณ ปัจจุบัน สามารถกดดำเนินการเพื่อทบยอดโทเค็นได้
- **Recent Activity**: ประวัติกิจกรรมล่าสุดที่เกิดขึ้นในระบบ เช่น คำขอลาได้รับการอนุมัติ หรือระบบทำการรีเซ็ตสิทธิ์
- **Team Pulse (Burnout Risk Index)**: แผนภูมิแท่งประเมินระดับความเหนื่อยล้าของคนในทีม เพื่อช่วยป้องกันความเสี่ยงในการทำงานหนักเกินไป

### 1.2 ปฏิทินปฏิบัติงานทีม (Team Shift & Token Calendar)
- **Shift & Token Calendar**: หน้าแสดงตารางปฏิทินแบบรายเดือนที่รวมวันหยุดและการสลับเวรปฏิบัติงาน
- **Date Cell (วันในปฏิทิน)**:
  - แสดงจำนวนสมาชิกที่ลางานในวันนั้นๆ พร้อมสัดส่วนการจำกัดความสามารถการลา เช่น `1/2` (มีคนลา 1 คน จากสิทธิ์ที่ลาได้สูงสุด 2 คน)
  - แสดงรายการสมาชิกในรูปแบบชิปสีระบุสถานะ เช่น ทำงานพิเศษวันหยุด (`+1 🚗`) หรือ วันหยุดทดแทน (`OFF`)
  - ล็อกอัตโนมัติเมื่อจำนวนคนลาถึงขีดจำกัดสูงสุด (Capacity Reached) หรือเป็นวันหยุดที่ตั้งค่าการล็อกไว้
- **Bank of Thailand Holidays**: บูรณาการวันหยุดธนาคารตามประกาศอย่างเป็นทางการของธนาคารแห่งประเทศไทย (BOT) ปี 2026 โดยจะล็อกวันดังกล่าวให้เป็นวันหยุดอัตโนมัติ (Public Holiday) และปิดไม่ให้มีการขอสิทธิ์ลาปกติ

### 1.3 ระบบโทเค็นและประวัติการเงิน (Token Balance Ledger)
- **Active Balance**: แสดงยอดกระเป๋าเงินโทเค็นคงเหลือของผู้ใช้งาน
- **How It Works**: คำชี้แจงกฎกติกาสิทธิ์การแลกโทเค็น
  - ทำงานในวันหยุดสุดสัปดาห์ (Weekend Shift) จะได้รับเพิ่ม 1 โทเค็น
  - การขอลาวันศุกร์หรือวันจันทร์ จะหักออก 3 โทเค็น
- **Transaction History**: รายการบันทึกประวัติการรับ-จ่ายโทเค็นอย่างละเอียด พร้อมสถานะการอนุมัติ (Approved / Pending)

### 1.4 รายชื่อบุคลากรและบทบาทหน้าที่ (Team Directory)
- จัดกลุ่มตามฝ่ายงาน (Management, Engineering, Design)
- การ์ดข้อมูลบุคลากรแต่ละคน (Member Card) แสดงรูปภาพ, บทบาทหน้าที่ (Team Lead, Admin, Member), ยอดโทเค็นสะสมของตนเอง และลิงก์ไปยังตารางเวลาของสมาชิกรายนั้นๆ
- ปุ่มควบคุมการดาวน์โหลดรายงานทีมงาน และการตั้งค่าสิทธิ์

### 1.5 การตั้งค่าระบบบริหารงาน (Settings Console)
- **Profile Settings**: ตั้งค่าชื่อผู้ใช้, รูปโปรไฟล์, และอีเมลติดต่อ
- **Notification Preferences**: จัดการเปิด-ปิดระบบแจ้งเตือนผ่านอีเมล (Email) และระบบข้อความด่วน (Push Notifications)
- **Workspace Settings**:
  - **Daily Capacity Limit**: แถบเลื่อนปรับสัดส่วนเปอร์เซ็นต์สูงสุดที่อนุญาตให้คนลางานพร้อมกันในวันเดียว
  - **Weekend Earn Rate**: เมนูเปลี่ยนอัตราตัวคูณในการให้รางวัลโทเค็นเมื่อทำงานในวันหยุด (1.0x, 1.5x, 2.0x)

## 2. โครงสร้างสถาปัตยกรรมและเทคโนโลยีที่เลือกใช้ (Architecture & Technologies)

- **Frontend & Routing**: Next.js App Router (เวอร์ชัน 16)
- **Styling**: Tailwind CSS (เวอร์ชัน 4) ร่วมกับ HeroUI สำหรับโครงสร้างเบสธีม
- **Design Aesthetic**: ธีมเดี่ยวสีขาวมินิมอล (Soft Off-White Theme) ออกแบบตามหลัก visual comfort เน้นความผ่อนคลายสบายตา ใช้โทนสี Slate/Zinc และการตัดขอบที่บางเบา
- **State Management**: บริการสืบค้นข้อมูลและดำเนินการประมวลผลบนฝั่งเซิร์ฟเวอร์ (SSR - Server Side Rendering) โดยใช้ URL Query Parameters ในการสลับปีและเดือนของปฏิทิน
- **Translation System (i18n)**: รองรับสองภาษา (ภาษาไทย TH / ภาษาอังกฤษ EN) ควบคุมผ่าน `LanguageContext` และมีตัวสลับภาษาบน Top Bar
